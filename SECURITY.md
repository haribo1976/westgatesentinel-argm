# Security Policy

## Reporting a Vulnerability

If you discover a security issue in this repository — including sensitive information accidentally committed, misleading content that could cause harm, or any other concern — please report it responsibly.

**Contact:** hello@westgatesentinel.co.uk  
**Subject line:** `[SECURITY] westgatesentinel-argm — <brief description>`

Please do not open a public GitHub issue for security concerns.

## What to Include

- A clear description of the issue
- The file(s) or content involved
- Why you consider it a security concern
- Any suggested remediation (optional)

## Response Commitment

We aim to acknowledge all reports within **3 working days** and resolve confirmed issues within **14 days** where possible.

## Scope

This repository contains documentation and framework content only — no application code, dependencies, or infrastructure configuration. The primary concerns are:

- Accidentally committed credentials or secrets
- Exposed personal data or client information
- Content that could be used for social engineering or prompt injection
